# Personal Portfolio Site Plan

## 参考站点提炼

我从 `logancyang/awesome-personal-websites` 里挑了几类代表做参考：

- Julia Evans: 导航清楚，内容按类别组织，适合让访问者快速找到项目、文章和背景。
- Lena Voita: 首页先给身份、研究方向、简历摘要和新闻，可信度建立很快。
- Eugene Yan: 把 writing、speaking、prototyping 分区，适合展示长期积累。
- Robert Hodgin / Vibert Thio: 作品型站点更重视大图、项目视觉和单个项目页面。
- Justin Pinkney: 有强烈个人风格，但结构仍然围绕作品和内容入口。

对你的求职站来说，最重要的不是把链接堆出来，而是把作品变成可见证据：界面、动图、可互动 Demo、项目复盘和结果数据。

## 页面信息架构

### 1. Hero

目标：让招聘方 30 秒内知道你是谁、想找什么岗位、能力方向是什么。

当前布局：

- 左侧：姓名、个人定位、主要 CTA、三个快速事实。
- 右侧：大型项目预览窗口，支持 Dashboard / Mobile / AI Tool 三种视觉切换。

后续替换：

- `Your Name`
- 个人介绍
- 目标岗位
- 真实项目截图或录屏

### 2. Portfolio Strategy Strip

目标：用三句话说明这个站点的阅读路径。

当前布局：

- 页面目标
- 核心表达
- 后续素材类型

后续可以改成：

- 你最强的三个卖点
- 例如「AI 产品原型」「前端交互实现」「数据驱动复盘」

### 3. Selected Work

目标：把项目做成可浏览的作品陈列区，而不是简历里的项目列表。

当前布局：

- 3 张精选项目卡片
- 每张卡片都有视觉预览区、项目标签、案例入口、播放预览按钮
- 支持产品 / 前端 / AI 筛选

后续替换：

- 项目封面图
- GIF 或 MP4
- Figma / GitHub / 线上 Demo 链接
- 真实项目简介和结果数据

### 4. Case Study

目标：展示一个项目的深度，说明你不是只做界面，而是能分析问题、推进方案、验证结果。

当前布局：

- 左侧：Problem / Role / Process / Result
- 右侧：可互动的用户旅程 Demo 区

后续替换：

- 重点项目的完整复盘
- 关键界面截图
- 用户流程图
- 可嵌入 Demo 或视频

### 5. Resume Snapshot

目标：让招聘方快速确认你的经历是否匹配岗位。

当前布局：

- 时间线
- 简历亮点面板
- PDF 简历入口

后续替换：

- 教育经历
- 实习 / 项目经历
- 奖项、证书、课程、研究经历
- PDF 简历

### 6. Capability Map

目标：把能力按招聘方关心的维度组织，而不是只列工具名。

当前布局：

- 产品判断
- 界面实现
- AI 应用
- 沟通协作

后续可以根据目标岗位调整，例如：

- 如果投产品：加强用户研究、指标、PRD、增长、商业分析。
- 如果投前端：加强 React、性能、可访问性、工程化、测试。
- 如果投 AI 应用：加强 LLM 工作流、RAG、评测、提示词、工具调用。

### 7. Contact

目标：让招聘方下一步动作明确。

后续替换：

- 邮箱
- GitHub
- LinkedIn
- 微信二维码
- 简历 PDF
- 面试预约链接

## 推荐素材优先级

1. 先给我 1 个最想展示的项目，包含截图、录屏和项目说明。
2. 再给 2 到 3 个补充项目，形成首页作品矩阵。
3. 给完整简历，我会拆成时间线和亮点面板。
4. 最后再微调视觉风格，让它更贴近你的个人气质和目标岗位。
